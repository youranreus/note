# Product Brief: Note

## 执行摘要

Note 是一款面向个人轻量记录场景的双态便签产品，提供“在线便签”和“本地便签”两条入口：用户既可以在未登录状态下快速输入 ID 直达记录页，也可以在登录后把便签沉淀为可管理、可收藏、可持续访问的个人资产。它要解决的不是复杂知识管理，而是用户在“突然要记一下”时常见的中断感、进入成本高和后续找回困难。

当前多数记录工具要么偏重编辑器能力，要么依赖账号体系先行，导致用户在最初几秒就被注册、创建文档、选择空间等动作打断。Note 的机会点在于把“立即记录”放在第一优先级，把“登录、收藏、归档、我的创建”作为自然升级路径，让用户先完成记录，再逐步进入账户价值闭环。

从产品形态看，Note 适合以极简、稳定、轻决策为核心心智切入。第一阶段重点不是做复杂协作，而是验证两件事：一是“低门槛入口”是否能显著提升首次使用和留存；二是“登录后资产承接”是否能让原本一次性的记录行为转化为可回访、可沉淀的个人内容库。

## 问题陈述

轻量记录场景里，用户的真实诉求通常非常简单：马上写下内容，稍后还能找回来。但现有方案存在三类明显摩擦。第一，进入门槛高。很多工具默认要求登录、创建页面或进入工作区，和“先记下来再说”的使用心理冲突。第二，记录与资产管理脱节。即便用户顺利写下内容，也常常缺少一个清晰路径把临时内容升级为长期可管理的资产。第三，跨场景使用不顺滑。有些内容只想本地保存，有些内容希望跨设备访问，但现有产品往往强迫用户在一开始就做平台或存储方式选择。

对用户而言，现状成本并不只是多点几次按钮，而是灵感流失、待办遗漏，以及无法建立稳定的“快速记录”习惯。对产品而言，这意味着新用户转化链路短、回访理由弱，很难形成长期价值。

## 解决方案

Note 的解决方案是围绕“先记录，后管理”重构完整体验。首页只保留最关键的决策：输入或生成 ID，选择在线便签或本地便签。在线便签负责跨设备访问、登录后归档、我的创建与我的收藏；本地便签负责极低成本的即时记录与本机保存。两者共享一致的界面语言和路径心智，降低用户学习成本。

登录被设计为能力增强而非使用前提。未登录用户依然可以进入核心流程；当用户需要跨设备访问、查看个人面板、收藏内容或管理自己的便签时，再通过 SSO 完成身份建立。登录完成后，产品通过“我的创建 / 我的收藏”把零散便签整理成可浏览、可回访的资产集合，形成从瞬时记录到长期管理的自然过渡。

在数据与安全层面，产品默认以稳定 ID 访问单条便签，并为受保护内容提供口令校验能力，避免敏感记录裸奔。同时通过唯一 SID、收藏幂等和用户主键体系，为后续扩展检索、分享和更复杂的权限能力打下基础。

## 差异化价值

Note 的差异化不在于功能堆叠，而在于体验顺序的取舍。它不是从“一个功能完整的笔记系统”往下裁剪，而是从“最低摩擦的记录入口”往上生长。相比传统在线文档工具，Note 更强调无须先做组织决策；相比纯本地便签，它又保留了登录后沉淀资产、管理创建与收藏的成长路径。

第二个差异点是双态并存但体验统一。在线和本地不是两个割裂产品，而是同一产品下针对不同记录意图的两种模式，既满足“马上记一下”的即时性，也兼顾“以后还要找”的可持续性。第三个差异点是登录价值明确，不把账号体系当作门槛，而是当作用户愿意升级时才出现的回报机制。

## 目标用户

主要用户是经常出现碎片化记录需求的个人用户，例如开发者、学生、内容工作者和高频信息处理者。他们通常不想在记录当下做过多设置，更在意打开即写、链接即达、稍后还能找到。

次级用户是已经形成轻量记录习惯、希望把零散内容逐步整理起来的人群。对他们来说，“我的创建”和“我的收藏”不是附加功能，而是决定产品能否从临时工具升级为日常工具的关键。

## 成功标准

- 首次访问用户能在极少步骤内进入便签页，首页到首条内容创建的转化率达到可持续增长。
- 在线便签入口使用率和本地便签入口使用率都能形成稳定分布，证明双态设计各自有明确需求承接。
- 登录后用户中，有显著比例会访问“我的创建”或“我的收藏”，表明资产管理能力被实际使用。
- 收藏、再次打开同一 SID、从首页回访历史便签等行为持续增长，说明产品不只是一次性工具。
- 核心稳定性指标达标：SID 唯一访问、SSO 回跳成功率、便签保存成功率和列表加载成功率保持在高水平。

## 范围界定

MVP 范围聚焦于四件事：极简首页入口、在线便签读写、本地便签读写、登录后个人面板（我的创建 / 我的收藏）。同时包含基础 SSO 登录回跳、单条便签稳定访问、收藏与取消收藏、必要的口令校验，以及支撑这些体验的数据结构重构。

明确不纳入首版范围的内容包括复杂协作、多级权限、富文本深度编辑、团队空间、复杂搜索和通知体系。首版目标是先验证“低门槛记录 + 登录后资产承接”这一核心价值是否成立，而不是过早进入重型知识管理竞争。

## 产品愿景

如果首版验证成功，Note 可以进一步发展成一个覆盖“即时记录、轻量沉淀、个人内容回访”的个人信息入口。中期可以沿着分享、检索、跨设备连续性和更细的内容组织方式演进；长期则有机会成为用户在正式知识系统之外、但高频打开的轻量捕捉层。

最理想的产品结果不是替代所有笔记工具，而是占据一个更高频、更轻决策的位置：当用户想到“先记一下”，第一反应就是打开 Note。
