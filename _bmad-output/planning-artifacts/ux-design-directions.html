note Design Directions

方向稿现已参考 docs/note.pen 的实际设计骨架：浅灰背景、右上状态 pill、 居中单列 shell、12px 圆角输入与 52px 主按钮、SSO 使用确认 modal、 登录回跳使用居中 loading card、个人中心使用 segmented tab modal。 下面的变化只在“主路径更像 Pastebin”还是“更忠于现有稿”之间做权重调整。

Direction 01 · Pen Baseline Home

直接继承 note.pen 首页：右上登录状态、居中 460px 壳体、ID 输入和两个入口按钮。

note 未登录

贴一段内容，拿一个固定地址

先开始，再决定是否登录、收藏或加编辑密钥。

输入 sid，或留空自动生成
结果页优先于管理页。用户按下主按钮后直接进入文本对象。
优点：最短路径风险：品牌感偏弱

Direction 02 · Baseline + Pastebin Emphasis

保留 note.pen 的骨架，只把主文案、状态提示和链接反馈做得更偏 Pastebin 主路径。

sid / n8a2j4d1 已保存
今天的同步说明如下：

1. 地址保持不变。
2. 之后补充内容时继续更新同一条。
3. 接收方打开仍看到最新版本。
优点：状态感稳风险：工具感更强

Direction 03 · Editor First Variant

仍然沿用单列居中结构，但让输入区更早成为主角，适合强调“先贴内容再说”。

自动生成 sid 可分享

从这里开始写

系统会自动为这段内容准备固定入口。

优点：内容中心明确风险：sid 价值表达较晚

Direction 04 · Result Object

结果对象感最强。用户创建后会明显感到“这是一条已经成立、可以维护的文本对象”。

/note/o/n8a2j4d1
固定地址已可访问，后续继续更新这里。
在线便签
创建者可编辑 实时状态：已同步
文本主体展示在对象卡片内，工具条围绕对象工作，而不是像后台页面那样分散布局。
优点：对象心智最清楚风险：首屏稍重

Direction 05 · Split Focus

左右分栏。左边内容，右边状态与增强能力。适合桌面，但移动要折叠。

文本内容

这里持续维护正文，右侧展示链接状态、权限和资产化动作。

Direction 06 · Asset Layer

保留轻量主流程，同时把“我的创建 / 我的收藏”作为一层弱管理面板附着在产品上。
